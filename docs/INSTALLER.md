# Windows installer

## Current setup

The installer is generated by:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\create_installer.ps1
```

It uses the one-file executable from:

```text
dist-onefile/TrackHunter.exe
```

and creates:

```text
release/TrackHunter-v2.2-Setup.exe
```

## User flow

The current setup executable:

1. Opens a graphical installer.
2. Shows the TrackHunter version.
3. Shows the Term of Use before installation.
4. Keeps `Continuar` disabled until the user checks the acceptance checkbox.
5. Lets the user choose the installation directory.
6. Installs `TrackHunter.exe`.
7. Creates `downloads`, `logs` and `state` folders under the selected installation directory.
8. Preserves an existing `state/tracklist.txt`.
9. Creates Desktop and Start Menu shortcuts.
10. Registers TrackHunter in Windows installed apps for the current user.
11. Copies `uninstall.ps1`.

## Uninstall behavior

The current uninstaller removes:

- selected installation folder;
- Desktop shortcut;
- Start Menu folder;
- current-user uninstall registry key.

Future installer work should add explicit uninstall choices to preserve or remove user data separately.

## Important limitations

- The setup is large, around 381 MB.
- The setup executable should not be committed to Git because GitHub blocks files larger than 100 MB.
- IExpress can briefly keep the generated setup locked after creation; the generator uses retry logic when copying to `release/`.
- The installer scripts are still embedded in `scripts/create_installer.ps1`; modularization is planned.

## Validation checklist

Before distributing a setup:

```powershell
python -m pytest -v
python -m PyInstaller --noconfirm --clean TrackHunterApp.spec
$env:PLAYWRIGHT_BROWSERS_PATH='0'; python -m PyInstaller --noconfirm --clean --onefile --windowed --name TrackHunter --icon assets\app_icon.ico --add-data "assets\logo.png;assets" --add-data "assets\app_icon.png;assets" --distpath dist-onefile --workpath build-onefile scripts\trackhunter_gui.py
powershell -ExecutionPolicy Bypass -File scripts\create_installer.ps1
```

Then manually validate:

- Term of Use appears first.
- `Continuar` is blocked until acceptance.
- installation folder selection works.
- app opens from the final button and shortcuts.
- app appears in Windows installed apps.
- uninstall removes the installed app.
